For each of the algorithms you wrote for problems 4-6, explain their time complexity and space
complexity using Big-O notation. Explain how you arrived at your answer.

•Problem 4:
	•Time Complexity:
		•The time complexity is O(n) where n = s.length() because the first loop runs n times and the second loop runs n times. 
			Thus as input size increases, program runtime increases linearly.
			The final complexity is calculated as O(n) + O(n) + O(1) = O(2n+1) = O(n)
	•Space Complexity:
		•The space complexity is O(n) where n = s.length() because creating the Stack of Strings named "stack" stores all characters from the input String.
			Making it of size n because it increases as s.length() increases. The rest of the program uses constant time.
•Problem 5:
	•Time Complexity:
		•The time complexity is O(n+m) where n = s.length() and m = output.length() because
			scanning the input String, s takes O(n) time and stack pushes and pops take constant time.
			Also, when a closed bracket "]" is reached, a for loop runs the sum of the digits within the String input.
			This comes to a final time of m to write to the output String.
	•Space Complexity:
		•The space complexity is O(n+m) where n = output because copying to the Stack, "stack" takes O(n) space and the String output can grow at O(m) space.
			Thus O(m) + O(n) = O(n+m).
